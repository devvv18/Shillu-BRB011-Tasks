== TOOLS ==
Maas = builds new server automatically via pipeline. Like ordering a pre-assembled computer.
Puppet = runs AUTOMATICALLY on new VM. Creates users, installs Java/Python, sets up SSH. PUPPET = sets up empty room.
Ansible = you run MANUALLY after Puppet. Installs ForgeRock DS software using playbooks. ANSIBLE = moves furniture in.
GitLab = code storage + pipelines. URL: https://prodgitlab.usaa.com
PasswordSafe = USAA secure vault. Cannot SSH directly from laptop to prod. Must go through PasswordSafe. URL: go/passwordsafe
Jump Server (prodfrctl01lsat) = middle server after PasswordSafe. You land here, then SSH to DS servers.
Jira = work tickets. MIAM-XXXXX format.
Prometheus = monitoring. stable variant = alerts ON (taking traffic). canary variant = alerts OFF (not yet live). New
servers start
as canary.
Canary Rollout = gradually shifts traffic 0%+5%>10%>100%. Roll back to 0% instantly if issues.
SSH = remote login to Linux. Uses key pair: private key (keep on laptop) + public key (register on GitLab/servers).
ServiceNow = IT request system. REQ = overall request. RITM = item under REQ. PasswordSafe needs RITM not REQ.
MGS (Metal Gear Solid) = team's infra automation platform. Wiki: go/mgswiki. MR approvals Slack: mgs-gitlab.
== 4 PHASES OF UPGRADE ==
Phase 1 - Deploy VMs: VMaas builds 4 servers. Puppet auto-configures. Isolated, no traffic. STATUS: DONE
Phase 2 - Install + Replicate: Ansible installs DS 8.0.2. Servers replicate each other. Still no live traffic. STATUS: IN PROGRESS
Phase 3 - Canary Rollout: Gradually shift traffic 0→100%. Monitor Prometheus. Rollback = set to 0%. STATUS: NEXT
Phase 4 - Retire Old VMs: Disable web port → wait for 0 connections → shutdown → remove from Prometheus → notify SecApps → decommission → update wikis. STATUS
