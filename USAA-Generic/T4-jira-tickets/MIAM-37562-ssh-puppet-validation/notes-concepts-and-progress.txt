BEFORE YOU START - WHAT IS ALL THIS? (Plain English)
What is a VM (Virtual Machine)?
Think of it like a computer that lives in the cloud/data center.
It's not a physical laptop - it's a "pretend" computer running
inside a big server at a USAA data center. We create these VMs
to host our software (ForgeRock Directory Services).
What is VMaas?
"VM as a Service" - it's a USAA internal tool/pipeline that
lets us ORDER and CREATE new VMs automatically by running a
pipeline (like pressing a button that builds the computer for you) .
We used this to create our 4 new prod servers.
What is Puppet?
Imagine you just bought a brand new laptop. It's completely empty -
no software, no users, no settings. You'd have to manually set
everything up.
Puppet does that AUTOMATICALLY on our VMs the moment they are created.
It's like a "setup script" that runs on its own and:
- Creates the dsadmin user account (so we can log in)
- Configures who is allowed to SSH into the server
- Installs base software like Python and Java
We don't have to do any of this by hand. Puppet reads a "recipe"
from the control-repo on GitLab and applies it to the server.
What is SSH?
SSH = Secure Shell. It's how you remotely log into a Linux server
from your laptop - like using Remote Desktop but for Linux, and
it uses a key (like a digital ID card) instead of just a password.
Without SSH access set up, you literally cannot do anything on th
1.00
What is an SSH Key?
Instead of a username + password to log in, Linux servers use a
"key pair":
- Private key → stays on YOUR laptop (never share this)
- Public key
→ you give this to the server / GitLab
(like giving someone a lock that only your key opens)
GitLab also uses your public key to verify it's really you when
you push/pull code.
What is Ansible? (for later tickets)
once the VM exists and we can SSH into it, Ansible is the tool
that actually INSTALLS the ForgeRock DS software on it.
Think of Puppet as "setting up the room" and Ansible as
"bringing in all the furniture."
Ansible uses "playbooks" - basically step-by-step instruction
files written in a simple format (YAML) that tell the server
what to install and configure.
What is GitLab?
GitLab is like Google Drive but for code. All our scripts,
pipelines, and configuration files (including Puppet recipes
and Ansible playbooks) live here. It also runs pipelines
(automated jobs) for us.
USAA's internal GitLab: https://prodgitlab.usaa.com
What is Jira?
Jira is the tool we use to track work. Every task is a "ticket"
or "story" with a number like MIAM-37562. You update the ticket
as you make progress and close it when done.
